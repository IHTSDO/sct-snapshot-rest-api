SNOMED CT Snapshot API v2

The SNOMED CT Snapshot REST API provides access to SNOMED CT content snapshot form (only the latest version for all components, see Snapshot definition in the RF2 format description).

This API has been developed by SNOMED International to support lightweight deployments of read-only SNOMED CT terminology servers.

Concepts ¶

Concepts allows access to SNOMED CT Concepts

Concept  ¶

A single Concept with all its details

Retrieve a Concept

Children  ¶

Lists all children of a concept, in the SNOMED CT hierarchy. Each child is retrieved in a summary form, called Concept Descriptor.

Retrieve Children
GET/v2/snomed/{edition}/{release}/concepts/{sctid}/children?form={form}

Example URI

GET http://browser.ihtsdotools.org/api/v2/snomed/en-edition/v20170731/concepts/123037004/children?form=inferred

HideShow

Identifier for the release, usually the release effective time preceded with a v.

sctid
number (required) Example: 123037004

SNOMED CT Identifier of the concept to retrieve.

form

Characteristic type of the Is-a hierarchy, either stated or inferred.

Members  ¶

Lists all reference set members of a concept. Each member is retrieved in a summary form, called Concept Descriptor. Only concepts that are reference sets will include members. The concept details describe all the reference sets where a concept is member in, called memberships, and can be retrieved with the /concepts/id call.

Retrieve Reference set members

Search  ¶

Performs a text search over the database, selecting mode and other limits. The search results include the matching descriptions plus a filter descriptor object, allowing to further refining the search by providing new limits.

Search descriptions
GET/v2/snomed/{edition}/{release}/descriptions?query={query}&searchMode={mode}&lang={language}&statusFilter={statusFilter}&skipTo={skipTo}&returnLimit={returnLimit}&normalize={normalize}&semanticFilter={semanticFilter}&moduleFilter={moduleFilter}&refsetFilter={refsetFilter}

Example URI

GET http://browser.ihtsdotools.org/api/v2/snomed/en-edition/v20170731/descriptions?query=asthma&searchMode=partialMatching&lang=english&statusFilter=english&skipTo=0&returnLimit=5&normalize=true&semanticFilter=finding&moduleFilter=900000000000207008&refsetFilter=450970008
URI Parameters
HideShow
edition
string (required) Example: en-edition

Identifier for the SNOMED CT Edition to browse, for example en-edition for the International Edition

release
string (required) Example: v20170731

Identifier for the release, usually the release effective time preceded with a v.

query
string (required) Example: asthma

String terms to use a query.

mode
string (optional) Example: partialMatching

Search mode, either partialMatching, fullText or regex.

language
string (optional) Example: english

Language for the search, only required for fullText mode.

statusFilter
string (optional) Example: english

Value to filter out undesired status, possible values are activeOnly, inactiveOnly or activeAndInactive.

semanticFilter
string (optional) Example: finding

Value to restrict search results to specific an semantic area, possible values are finding, disorder, procedure, etc.

moduleFilter
string (optional) Example: 900000000000207008

Value to restrict search results to an specific module, possible values are any valid ModuleId.

refsetFilter
string (optional) Example: 450970008

Value to restrict search results to an specific reference set, possible values are any valid Reference Set conceptId.
